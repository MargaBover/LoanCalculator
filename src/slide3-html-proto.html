Design principles in practice:

  • Divide and Conquer:

    Accross this project the different stylesheet files were used for the different parts of the project.

  • Design for portability:

    Any non-widely supported HTML / CSS feature by the main browsers was avoided.

  • Increase Reusability:

    The style was separated from the HTML code and a naming prefix was used to avoid name clashing in future uses.

  • Documentation and collaboration:
    • The code was tracked using git
    • The classes and ids naming was done comprehensively
